During the implementation of this assignment, I encountered a few key challenges that helped me understand the importance of proper error handling and database design. In commit e162894 (Add logging middleware and database-driven login system), I initially struggled with integrating the logging middleware correctly because I wasn't appending newlines to the log.txt file, which caused all log entries to run together on one line. After debugging this issue and fixing it, I realized how critical it is to test file I/O operations thoroughly. Another significant challenge came in commit 2b62d86 (Add bcrypt password hashing and storage) when I had to refactor the password comparison logic—I initially tried to compare plaintext passwords directly with the hashed ones, which obviously didn't work. Learning to use bcrypt.compare() properly taught me the importance of understanding how cryptographic libraries work before implementing them. The experiment of cascading deletes in the deleteUser() function was particularly useful; when I delete a user, all their articles are automatically removed from the database, which prevents orphaned data and maintains referential integrity. Overall, this assignment reinforced the importance of planning database operations carefully and testing each component individually before integrating them together.
