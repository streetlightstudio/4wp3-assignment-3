While implementing this application, I encountered two notable challenges that required debugging and experimentation. The first issue arose during the initial setup of the login and logging systems (commit e162894). At that stage, the login controller was querying the Users table and comparing passwords directly, but since bcrypt had not yet been integrated, the password comparison logic needed to be restructured once hashing was added in a later commit. This taught me the importance of planning the full authentication flow — including hashed storage and comparison — before wiring up the login route, rather than adding security incrementally. The second issue was a syntax error in the signup form handling that was caught and fixed in commit fdc3879. The router.post handler for the signup route had a structural mistake in how the conditional branches were written, causing the server to throw an error when the signup form was submitted. Stepping through the logic and carefully checking bracket placement and the async/await flow resolved the issue. This experience reinforced how easy it is to introduce subtle syntax errors in nested async route handlers, and highlighted the value of testing each route immediately after writing it rather than waiting until all features are implemented.
