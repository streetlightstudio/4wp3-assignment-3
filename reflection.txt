During the implementation of this assignment, I encountered a few key challenges that helped me understand the importance of proper error handling and database design. In commit 92f34e3 (Add input validation and trimming to signup form), I implemented input validation that trims whitespace from user inputs, which ensures consistent database entries and prevents edge cases with accidentally added spaces. Another significant challenge came from learning how bcrypt works properly; when I initially struggled with password hashing, I realized the critical importance of using proper cryptographic libraries instead of plaintext storage. Adding error handling with try-catch blocks in commits like f623688 (Add error handling with try-catch blocks to user model functions) and 229774e (Add error handling to article model database operations) taught me that database operations can fail in many ways - connection issues, constraint violations, or data integrity problems. By wrapping these operations in error handlers, the application now fails gracefully and logs issues instead of crashing. Improving the UI with placeholders, required attributes, and better labels (commits 9e76091, 5e71746) demonstrated how small UX improvements can make the application more user-friendly and guide users to input valid data. Overall, this assignment reinforced that good software is built through iterative improvements, comprehensive error handling, and always thinking about the user experience.
